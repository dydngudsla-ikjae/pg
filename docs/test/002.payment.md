# 결제 서비스 테스트 시나리오

## 결제 승인 (POST /v1/payments)
- [ ] 결제 승인 API가 카드 결제 유효한 요청으로 200과 PAID 상태를 반환한다
- [ ] 결제 승인 API가 paymentKey를 pay_ 접두어 형식으로 생성한다
- [ ] 결제 승인 API가 카드 원문(number, expiry, birthOrBizNo, pwd2digit)을 DB에 저장하지 않는다
- [ ] 결제 승인 API가 cardCompany, cardLast4, maskedCardNumber를 DB에 저장한다
- [ ] 결제 승인 API가 승인 후 outbox_event(payment.paid)를 PENDING 상태로 저장한다
- [ ] 결제 승인 API가 금액 1,000만원 요청 시 승인한다
- [ ] 결제 승인 API가 금액 1,000만원 초과 요청 시 400을 반환한다
- [ ] 결제 승인 API가 orderId 누락 시 400을 반환한다
- [ ] 결제 승인 API가 amount 누락 시 400을 반환한다
- [ ] 결제 승인 API가 method 누락 시 400을 반환한다
- [ ] 결제 승인 API가 CARD 방식에서 card 정보 누락 시 400을 반환한다
- [ ] 결제 승인 API가 Idempotency-Key 헤더 누락 시 400을 반환한다
- [ ] 결제 승인 API가 PG 거절 시 status=FAILED로 저장하고 실패 응답을 반환한다
- [ ] 결제 승인 API가 PG 타임아웃 시 status=UNKNOWN과 함께 응답한다
- [ ] 결제 승인 API가 동일 가맹점의 orderId 중복 요청 시 실패한다

## 멱등성 (Idempotency-Key)
- [ ] 동일 멱등키로 완료된 요청을 재요청하면 캐싱된 동일 응답을 반환한다
- [ ] 동일 멱등키이지만 request body가 다르면 422를 반환한다
- [ ] 동일 멱등키로 처리 중인 요청이 있으면 409를 반환한다
- [ ] 결제 승인에 사용한 멱등키로 결제 취소를 요청하면 별개 요청으로 처리된다

## 결제 조회 (GET /v1/payments/{paymentKey})
- [ ] 결제 조회 API가 paymentKey로 200과 전체 필드를 반환한다
- [ ] 결제 조회 API가 존재하지 않는 paymentKey 요청 시 404를 반환한다
- [ ] 결제 조회 API가 부분취소된 결제 조회 시 cancelledAmount와 remainAmount를 포함한다

## 결제 목록 조회 (GET /v1/payments)
- [ ] 결제 목록 조회 API가 200과 페이징 정보를 반환한다
- [ ] 결제 목록 조회 API가 status 필터로 특정 상태의 결제만 반환한다
- [ ] 결제 목록 조회 API가 from/to 날짜 필터를 적용한다
- [ ] 결제 목록 조회 API가 다른 가맹점의 결제를 반환하지 않는다

## 결제 취소 (POST /v1/payments/{paymentKey}/cancel)
- [ ] 결제 취소 API가 전액 취소 후 status=CANCELLED와 200을 반환한다
- [ ] 결제 취소 API가 동일 멱등키로 완료된 취소 재요청 시 캐싱된 동일 응답을 반환한다
- [ ] 결제 취소 API가 부분 취소 후 status=PARTIAL_CANCELLED, cancelledAmount, remainAmount를 반환한다
- [ ] 결제 취소 API가 부분취소 여러 번 시 누적 취소액을 합산한다
- [ ] 결제 취소 API가 부분취소로 잔여금액이 0이 되면 CANCELLED로 전이한다
- [ ] 결제 취소 API가 누적 취소액이 원금 초과 시 400을 반환한다
- [ ] 결제 취소 API가 CANCELLED 상태 결제 취소 시 400을 반환한다
- [ ] 결제 취소 API가 FAILED 상태 결제 취소 시 400을 반환한다
- [ ] 결제 취소 API가 READY 상태 결제 취소 시 400을 반환한다
- [ ] 결제 취소 API가 cancelAmount 누락 시 400을 반환한다
- [ ] 결제 취소 API가 Idempotency-Key 헤더 누락 시 400을 반환한다
- [ ] 결제 취소 API가 merchantCancelId 중복 요청 시 422를 반환한다
- [ ] 결제 취소 API가 존재하지 않는 paymentKey 요청 시 404를 반환한다
- [ ] 결제 취소 API가 취소 후 outbox_event(payment.cancelled)를 PENDING 상태로 저장한다

## 결제 결과 확정 — 내부 (POST /internal/payments/{paymentKey}/verify)
- [ ] verify API가 X-Internal-Service 헤더 없이 호출 시 401 또는 403을 반환한다
- [ ] verify API가 UNKNOWN 상태 결제를 PAID로 확정한다
- [ ] verify API가 UNKNOWN 상태 결제를 FAILED로 확정한다
- [ ] verify API가 READY 상태 결제를 PAID로 확정한다
- [ ] verify API가 READY 상태 결제를 FAILED로 확정한다
- [ ] verify API가 UNKNOWN 확정 후 outbox_event를 PENDING 상태로 저장한다
- [ ] verify API가 PAID 상태 결제에 호출 시 상태 변경 없이 200을 반환한다
- [ ] verify API가 존재하지 않는 paymentKey 요청 시 404를 반환한다
