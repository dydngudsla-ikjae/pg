# 가맹점 서비스 테스트 시나리오

## 가맹점 등록 (POST /v1/merchants)
- [x] 가맹점 등록 API가 유효한 요청으로 201을 반환하고 ACTIVE 상태로 생성한다
- [x] 가맹점 등록 API가 merchantNo를 M+yyyyMMdd+일련번호 형식으로 생성한다
- [x] 가맹점 등록 후 조회 시 등록 데이터가 일치한다
- [x] 가맹점 등록 API가 name 누락 시 400을 반환한다
- [x] 가맹점 등록 API가 businessNo 누락 시 400을 반환한다
- [x] 가맹점 등록 API가 representativeName 누락 시 400을 반환한다

## 가맹점 조회 (GET /v1/merchants/{id})
- [x] 가맹점 조회 API가 모든 필드를 포함하여 200을 반환한다
- [x] 가맹점 조회 API가 존재하지 않는 id로 요청 시 404를 반환한다

## API 키 발급 (POST /v1/merchants/{id}/api-keys)
- [x] API 키 발급 API가 env=live 요청 시 mk_live_ 형식의 원문 키와 함께 201을 반환한다
- [x] API 키 발급 API가 env=test 요청 시 mk_test_ 형식의 원문 키와 함께 201을 반환한다
- [x] API 키 발급 API가 동일 가맹점에 여러 키 발급을 허용한다
- [x] API 키 발급 후 DB에는 원문 키가 저장되지 않고 key_hash만 저장된다
- [x] API 키 발급 API가 존재하지 않는 가맹점 id로 요청 시 404를 반환한다
- [x] API 키 발급 API가 유효하지 않은 env 값으로 요청 시 400을 반환한다
- [x] API 키 발급 API가 env 누락 시 400을 반환한다

## API 키 폐기 (DELETE /v1/merchants/{id}/api-keys/{keyId})
- [x] API 키 폐기 API가 keyId와 revokedAt을 포함하여 200을 반환한다
- [x] API 키 폐기 API가 존재하지 않는 keyId로 요청 시 404를 반환한다
- [x] API 키 폐기 후 해당 키로 검증 API를 호출하면 valid=false, reason=REVOKED를 반환한다

## 웹훅 URL 관리 (PUT /v1/merchants/{id}/webhook)
- [x] 웹훅 URL 수정 API가 URL을 저장하고 200을 반환한다
- [x] 웹훅 URL 수정 API가 잘못된 URL 형식으로 요청 시 400을 반환한다
- [ ] 웹훅 URL 수정 API가 존재하지 않는 가맹점 id로 요청 시 404를 반환한다

## 가맹점 상태 변경 (PATCH /v1/merchants/{id}/status)
- [ ] 가맹점 상태 변경 API가 SUSPENDED로 변경 후 200을 반환한다
- [ ] 가맹점 상태 변경 API가 SUSPENDED에서 ACTIVE로 복원을 허용한다
- [ ] 가맹점 상태 변경 API가 유효하지 않은 status 값으로 요청 시 400을 반환한다
- [ ] 가맹점 상태 변경 API가 존재하지 않는 가맹점 id로 요청 시 404를 반환한다

## API 키 검증 — 내부 (POST /internal/api-keys/verify)
- [ ] 검증 API가 X-Internal-Service 헤더 없이 호출되면 401 또는 403을 반환한다
- [ ] 검증 API가 유효한 키 + ACTIVE 가맹점 요청 시 valid=true, merchantStatus=ACTIVE를 반환한다
- [ ] 검증 API가 유효한 키 + SUSPENDED 가맹점 요청 시 valid=true, merchantStatus=SUSPENDED를 반환한다
- [ ] 검증 API가 존재하지 않는 키로 요청 시 valid=false, reason=NOT_FOUND를 반환한다
